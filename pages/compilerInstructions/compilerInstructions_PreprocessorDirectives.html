Compiler Instructions

What Are Preprocessor Directives?

Preprocessor directives in C are any keywords with an octothorp (#) in front. They are special instructions meant for the preprocessor part of the compiler to perform, before the code is actually compiled.

Compilation Process Review

Compiling Code is performed in four steps.

  1. Preprocessing
  2. Compilation
  3. Assembly
  4. Linking

Preprocessor directives like "#define" (and most other keywords beginning with an #octothorp) are handled during the preprocessing step, before the code reaches the "actual" compiler. It can be confusing, because "Compiler" can refer generically to all the programs that perform all the steps above, or specifically to the program that does the Compilation step.

The C Preprocessor Directives

The following are all the C Preprocessor directives, their syntax, and notes on their use.

#Define

#define defines a "shorthand" for another chunk of code. The official name for this is a macro. When the preprocessor encounters a #define, it makes a note of the name, or identifier, and the code that follows. Then, anyplace else in the code the identifier is encountered, the preprocessor replaces the identifier with the chunk of code associated with it, cutting and pasting the longer code in to replace the shorthand.

#define example


#define INIT_VALUE 10
#define PRINT_SOME_STARS printf("***")


int main(void)
{
	int myVariable = INIT_VALUE;
	//INIT_VALUE gets replaced with "10"
	
	PRINT_SOME_STARS; 
	//gets replaced with printf("***")"
	
	myVariable++;
	
	return 0;
}

#ifdef

The #ifdef keyword is used with the #endif keyword to surround of code. If the statement after the #ifdef is defined elsewhere as a #define, the preprocessor includes the code in the block. Otherwise, the code is cut out. this is called Conditional Compilation, since its a conditional statement that controls if the code is compiled or not. 


#define STARS "***"

int main(void)
{

    #ifdef STARS
        //Doesn't get compiled if 
        //"#define STARS" is missing
        print(STARS);
    #endif
	
    return 0;
}

#ifndef

#ifndef checks if the next statement is NOT defined in the code as a #define.

#else

The preprocessor directive equivalent of an ordinary else statement. It is used with the #ifdef and behaves as you would expect.

Preprocessor Directive Gotchas

Preprocessor directives never reach the compiler proper. By the time the compiler sees the code, it has been chopped and swapped and flipped around. Most debuggers will throw a fit if an error is encountered in a chunk of code that a #define swapped in. In fact most debuggers won't even try.

Be aware that preprocessors are shifting the code around in ways the compiler and debugger don't necessarily expect, and this can lead to strange errors and erratic debugger behavior if you are not careful.
