Compiler Instructions

Static Variables

The static keyword indicates that a given variable should only be stored in one fixed place in memory, and not on the stack.

What this means is that static variables exist all the time. Even if defined in a function, static variables still exist when the function exits. If the same function accesses the variable next, it will have whatever data it had when the function last exited. It will also retain its data even if another function accesses the variable.

An Ordinary Variable in Memory

When a function begins, a small chunk of memory is allocated for it on the stack.

Later in the function, when a variable is defined, another chunk of memory is allocated for it on the stack.

Finally, when the function ends, or exits scope, all space allocated for the function and variables it declared are unallocated.

A Static Variable in Memory

When a static variable is declared, regardless of in a function or in main, it is allocated a piece of memory outside of the stack, making it persistent, regardless of what function is currently being run.
