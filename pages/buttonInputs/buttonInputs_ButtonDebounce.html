Button Debounce

Buttons wired into a microcontroller for digital reading experience a phenomenon called Button Bounce . This is when upon being pressed, the button does not close all at once. Instead, the button "bounces" between an open and closed state before fully transitioning to a closed state.

If a microcontroller reads the button state in the middle of this transition period, it is uncertain what the read result will be. In order to fix this, all button reads should be Debounced

How to De-bounce Button Reads

To debounce, simply read the button twice, with a fixed waiting time between reads that is greater than the rated bounce time (typically given by the manufacturer).

If the two reads agree, the observed state is "stable" and the read is good. If the two reads disagree, the observed state is "unstable", or perhaps the result of electrical noise. The read data should be ignored.
