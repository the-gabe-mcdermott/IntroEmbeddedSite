Bit Manipulation

Bitwise Operators

Bitwise operators are just like ordinary mathematical operators (+, -, etc.) They take two numbers as input (also known as two operands ), and do something to them to create a result.

This section will provide an overview of all the bitwise operators available in C. The next section will elaborate on using bitwise operators with masks to set or clear bits, using an example.

C Boolean Logic Comparison Operators

There are three C operators for doing Boolean logic comparisons between two numbers, bit by bit. For example, the OR operator, written with the syntax "C = A | B;"�, takes the first bit in variable A and the first bit in variable B, and performs a logical OR on the two bits. The result is stored in the first bit of C. Then, the second bit of A is compared against the second bit of B, and the result stored in the second bit of C. The same for the third bit, fourth, and so on.

Operator Name C Syntax
OR A | B
AND A & B
EXCLUSIVE OR A ^ B

Or Operator Example


	A = 0011;
	B = 1010;

	C = A | B; 
	//C => 1011

And Operator Example


	A = 0011;
		
	B = 1010;

	C = A & B;
	//C => 0010

Exclusive Or Example


	A = 0011;
	B = 1010;
	
	C = A ^ B;
	//C => 1001

C Shift Operators

Shift operators shift the bits in the first operand by the number of positions given in the second operand. There are left shift and right shift operators.

You might be wondering what happens to the bits at the furthest right and left of the number. In a left shift, the left most bit is lost, and the right most bit gets assigned a zero. In a right shift, the right most bit is lost, and the left most bit gets assigned a zero.

Operator Name C Syntax
LEFT SHIFT A << B
RIGHT SHIFT A >> B

Left Shift Example


	A = 0110;
	B = 1;
	
	C = A << B;
	//C => 1100;
	//Additional shifts cause bits to "roll off"�
	C = C << B;
	//C => 1000;

Right Shift Example


	A = 0110;
	B = 1;

	C = A >> B;
	//C => 0011;
	//Additional shifts cause bits to "roll off"�
	C = C >> B;
	//C => 0001;

The C Negation Operator

Negation is performed as a unary operation. That is, it only has one operand. The bits in the operand are all flipped to their opposite.

Operator Name Operator Synatx
ONES COMPLIMENT aka NOT ~A

Ones Compliment Example

	
	A = 0011;
	
	C = ~A;
	//C => 1100;

C Bitwise Assignment Operators

Assignment Operators also exist, which assume that the variable on the left hand side is both the first operand, and also the variable to store the result in. The Bitwise Assignment Operators Are:

Operator Name Operator Syntax
OR EQUALS |= A
AND EQUALS &= A
EXCLUSIVE OR EQUALS ^= A
LEFT SHIFT EQUALS >>= A
RIGHT SHIFT EQUALS <<= A

Or Equals Example


	A = 0011;
	B = 1010;
	
	A |= B;  
	//A => 1011;

And Operator Example


	A = 0011;
	B = 1010;

	A =& B;
	//C => 0010

Exclusive Or Example


	A = 0011;
	B = 1010;

	A =^ B;
	//A => 1001

Left Shift Example

	A = 0110;
	B = 1;

	A =<< B;
	//A => 1100;
	//Additional shifts cause bits to "roll off"�
	A =<< B;
	//A => 1000;

Right Shift Example


	A = 0110;
	B = 1;

	A =>> B;
	//A => 0011;
	//Additional shifts cause bits to "roll off"�
	A =>> B;
	//A => 0001;
